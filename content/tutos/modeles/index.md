---
title: "Créer des modèles"
date: 2025-11-07
---

La création de modèles (ou *templates*) permet un véritable gain de temps pour normaliser la création de note. Ainsi, les notes de **réunions**, de **formations** ou des **fiches de lecture** présenteront les mêmes champs de métadonnées à remplir, les mêmes titre etc.

### 1- Créer la note qui servira de modèle

Créer une note intitulée « Réunion ». Renseigner les métadonnées qui vous semble utiles, pour pouvoir les interroger avec le module Bases.

<img src="/img/modele_1.gif">


### 2- Renseigner le corps de la note

Ajouter les titres qui structureront votre note.

<img src="/img/modele_2.gif">

### 3- Déplacer la note créée

La note qui vient d'être créée doit être déplacée dans un dossier, dont l'intitulé est libre. Je vous conseille de le nommer `Modèles` ou `Templates`, pour qu'il soit identifié comme étant le dossier qui accueillera tous vos modèles.

<img src="/img/modele_3.gif">

### 4- Paramétrer Obsidian

Il faut maintenant indiquer à Obsidian où se trouve votre dossier de modèles, pour qu’il puisse chercher les notes à incorporer. Dans les `Modules principaux`, cliquer sur `Modèles`, et renseigner le bon dossier.

<img src="/img/modele_4.gif">

### 5- Charger le modèle dans une nouvelle note

Toujours dans l’exemple d’une réunion, créer une nouvelle note. Ouvrir la console de commande (avec l’interface graphique ou `Ctrl+P`), et rechercher la fonction `Modèles : insérer un modèle`.

<img src="/img/modele_5.gif">

Si plusieurs modèles existent, choisissez celui que vous souhaiter charger, et cliquez.

<img src="/img/modele_6.gif">

La note nouvellement créée comporte maintenant tous les attributs du modèle. Vous pouvez modifier les métadonnées et prendre vos notes.
